System Architecture · Design Exploration

DIREKTA

You direct. AI delivers.

A browser-based film studio that turns a script into consistent, shootable frames — by pinning identity as trained models and look as fixed text, then treating every shot as a new camera on locked things. This document maps how the pipeline works, end to end.

The core idea

Direkta never asks the AI to improvise consistency. Identity (who / where) is frozen as trained Soul-ID models; look (the grade) is frozen as the Bible's visual-language text, injected byte-identical into every prompt. Once both are locked, a "shot" is just those locked things viewed from a new angle.

02 · Control Model

The agent never goes off the leash

This is not an autopilot you launch and watch. It is a guided sequence where the director is the clutch between every step: the agent does one step, proposes, and stops — you review, edit, commit, then proceed.

Sequential & gated

Walk, don't run

Workspaces unlock in order. Neither you nor the agent skips ahead — the locks already exist in page.tsx.

Propose · don't commit

You hold the pen

Agents write into proposals, never final tables. Clarifications surface only for forks that cascade expensively.

"Depends on the stage"

Friction matches stakes

Cheap text is drafted for you to edit; anything that spends money pauses per-item for your explicit go.

03 · Scripting

The writing front-end

Not five tabs — one home (Draft) with contextual affordances, governed by a single rule: be in the cheapest surface that resolves your current doubt.

Shape?

Outline

A collapsible left gutter — acts → sequences → scene cards. Your plan, not the analyst's beats.

Words?

Draft

The Fountain script. The only surface that writes projects.script — and only two ways: you type, or you accept a proposal.

Reference?

Codex

Your living notes (characters, world, rules). Auto-proposed one-line cards from the live parse — curate by exception.

Is the idea good?

Discuss

A story-room chat that interrogates the premise rather than drafting it. Writes nothing.

Does it work?

Analyse

On-demand coverage notes — quote-or-suppress, internal-consistency focused. Feedback, never breakdown.

Anti-slop

Cold-start voice

The agent won't draft the first scene cold. You supply a tone card, a style sample, or scene one by hand. Voice anchors to your reference, never genre defaults.

Codex vs Movie Bible

The #1 confusion risk, resolved decisively: Codex = your intent, written forward. Bible = the agent's reading, extracted backward. Same field-shape, opposite direction, opposite sides of the submit gate. They never merge.

CodexMovie Bible
AuthorYou, by hand (agent may propose)Waved POV passes, automatically
WhenWhile writing · submitted = 0After submit · submitted = 1
Truth basisIntent — "how I want it"Evidence — "the script says, quote: …"
DirectionInput to writingOutput of analysis
At the seamFed as priors / prompt-context — alias resolution, genre disambiguation, attention bias. Never pre-written into a Bible column.Every claim must carry a verbatim script quote, or it is rejected.
After submitThe Codex becomes the reconciliation yardstick — each Bible item tagged confirmed / divergent / unsupported (a hole) / emergent. The gap between your intent and what you actually wrote is the single most valuable signal in the tool.
Defect this design fixes

Today, "Edit script" flips script_submitted to false and silently throws the entire breakdown away. The replacement freezes a draft snapshot on submit; the breakdown always reads from that snapshot (never live script); a re-edit raises a propose-don't-commit drift banner — and a script edit never auto-invalidates a trained Soul ID.

The anti-slop gate

A schema enforces shape, not insight. So interpretive fields (psychology_want, themes, voice) carry an evidence contract: every claim cites a scene + a verbatim quote, and Supervision rejects any claim whose quote isn't literally in the script. No quote, no claim.

Single source of truth

The Bible is the root of a fingerprint dependency graph. Change a scene (hashed by body, not heading) and only its downstream Bible sections, beats, and frames get stale-flagged for regeneration — never silently overwritten.

05 · Casting · The Soul-ID Subsystem

Train first — then every frame is consistent by construction

A character starts as text only. Before anything renders, Casting trains a Soul ID for every character and every location. Consistency stops being a per-frame gamble and becomes a property of the models. Casting is a hard gate in front of the Cinematographer.

The bootstrap

Text → a trainable set

You have no photos. So: roll faces → lock the one that is the character → expand that single anchor into a consistent set with a face-identity model → that set is what the LoRA trains on. This is the step that defeats drift.

Hybrid engine

Identity model + Fal LoRA

A face-identity model builds the consistent training set; a Fal Flux LoRA is then trained for render-time control with a unique trigger word. Best fidelity + look control, stays in your existing Fal infra.

The crux — multi-LoRA composition

Two characters in a room = three trained LoRAs in one render (char A + char B + location), and stacking trained identities is where faces bleed. Strategies under design: tuned per-LoRA scales, a hero-LoRA + secondary-by-region, or render-the-room-then-inpaint. A decision rule caps how many LoRAs are safe in one pass, with a fallback when a beat exceeds it. — being hardened in the running design pass.

Locations — the riskier half

LoRAs memorise views, not 3-D space — great for faces, harder for a room that must hold across a wide, a reverse, and a CU. You chose to train locations too; if it proves shaky, the fallback is a reference-plate per location while keeping full training for faces.

06 · The Cinematographer Agent

Invoke the locks — don't describe them

With identity trained and look fixed, rendering a beat is composition, not a gamble. The prompt is built from five ordered layers; identity is carried by the LoRAs, the grade by the Bible.

1
FramingShot size · lens · height · movement — the only layer that varies across a coverage set.
2
Subjects + Soul IDsTrigger words + LoRA refs for each character present. Identity is guaranteed by the model, not the words.
3
SettingLocation Soul ID + int/ext + time of day + the beat's props.
4
Bible visual language — the look-lockPalette hexes · lighting philosophy · cinematography notes · motifs. Byte-identical for every frame in the film.
5
Mood + format tailThe beat's mood + aspect-ratio framing hint + quality tail.

Why multishot falls out for free

Because the LoRA holds identity across any angle, a coverage set is just "same LoRAs + same look, vary only the framing." That reads as the same instant from different cameras. Heroes-only by default (one establishing + one hero angle per beat) keeps the bill sane; the full set is opt-in per beat, behind a pre-render cost estimate.

Known repo constraints (design notes, not changes)
  • projects.update has a closed Pick<> whitelist — new state lives in a side table.
  • bible.update silently drops JSON/array values — needs a JSON branch before stale_sections round-trips.
  • skillFor(layer) selects by layer only — needs a variant axis so a story skill doesn't shadow the canvas pipeline.
  • /tmp SQLite resets on cold start — the reason Neon + Blob is a prerequisite, not a later nicety.

08 · Decisions Locked This Session

What the director chose

The forks that shaped the architecture above, with the call you made.

Control
Sequential, user-gated, propose-don't-commit. The agent never auto-advances; you are the clutch between every step.locked
Step size
Depends on the stage. Cheap text is drafted for you to curate; expensive / identity actions pause per-item.locked
Consistency
Train-first Soul IDs for characters and locations. Nothing renders until identities are trained.locked
Engine
Hybrid — face-identity model builds the training set, Fal Flux LoRA trains for render control.locked
Storage
Neon + Blob first. Durable LoRA weights, training sets, and run state — migrated before the subsystem ships.locked
Coverage
Heroes-only by default. One establishing + one hero angle per beat; the full set is opt-in behind a cost estimate.locked
Codex
Seed, then reconcile — never merge. Fed to breakdown as priors; becomes the intent-vs-reading yardstick after submit.locked

Still open (recommended defaults)

  • First-run lane — offer both, make the express "just draft me a rough cut" lane prominent.
  • Inline ghost text — off by default, per-director toggle (most steamroll-prone surface).
  • Re-edit granularity at MVP — whole-script re-break-down behind a drift banner; surgical per-scene as the first post-MVP increment.
  • Import scope — include .fdx (cheap XML→Fountain), exclude .pdf (needs OCR-grade cleanup).
  • Multi-LoRA composition — the safe-LoRA-count rule and fallback, being hardened now.
